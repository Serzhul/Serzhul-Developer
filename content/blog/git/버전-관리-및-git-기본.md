# 버전 관리란?

- 두 명의 개발자가 하나의 서비스를 만드는 과정
  - 각자 이름을 폴더를 만들고 각자 개발하다 메일로 보내서 합친다.
  - 에러가 발생할 수 있으니 백업본을 만든다.
  - 상대방이 작업중인 파일을 고치려면 메일에 따로 적어준다.
  - 코드를 합칠 때 확인해서 내 코드에도 반영
- 위와 같은 과정을 쉽게 도와주는 프로그램이 버전 관리 프로그램

# Git

- 버전 관리 프로그램의 하나로, 최근에 가장 많이 사용되고 있는 버전 관리 종류라 할 수 있다.
- 리눅스의 아버지인 리눅스 토발즈가 2005년 처음 개발했다.
- 2021년 8월 16일에 2.33 버전이 출시되었다.

## Git을 사용하는 두 가지 방법

- CLI
  - Command Line Interface
- GUI
  - Graphic User Interface

## Git 호스팅 사이트

- git으로 관리한 코드들을 호스팅 할 수 있도록 도움을 주는 사이트들을 말한다.

### Github

- 모기업은 Github Inc로 최근 MS사가 인수한 바 있다.
- 사용자 2,800만명의 명실상부 세계 최대 규모 Git 호스팅 사이트이다
- 공개 저장소 생성 무료, 비공개 저장소는 작업자 3인 이하인 경우 무료, 설치형 버전인 Enterprise는 월 21달러

### GitLab

- 모기업은 GitLb Inc이다
- NASA, Sony 등 10만개 이상 조직이 사용중이며 GitLab 자체가 오픈소스인 것이 특징이다.
- 공개, 비공개 저장소 생성이 무료이다.

### BitBucket

- 모기업은 Atlassian이다.
- 사용자 600만명, 이슈 추적 툴인 지라(Jira)와 연동이 쉽다.
- 5명 이하 팀이면 공개 및 비공개 저장소 생성이 무료다.

# Github에 코드를 올리는 과정

1. 로컬 프로젝트 폴더에 git을 명령 (`git init`)
2. 코딩
3. 변경한 파일 중 올리길 원하는 것만 선택 (`git add`)
4. 선택한 파일들을 하나의 버전(commit)으로 만들고 설명(comment) 적기 (`git commit -m '~~'`)
5. github 사이트에 프로젝트 저장소 만들기
6. 로컬과 github 저장소 연동 (`git remote add`)
7. github에 올리기 (`git push`)

# 1. git init

- 제일 처음 로컬 프로젝트 폴더에 해당 폴더에 git을 사용하겠다고 선언하는 명령어이다.
- git init을 실행하고 나면 해당 폴더에 .git이라는 숨김 폴더가 생성되며, .git이 존재하는 폴더를 로컬 저장소라 부른다.
- 로컬 저장소에는 버전 정보, 원격 저장소 주소 등이 저장된다.
- 원격 저장소에서 로컬로 코드를 받아오면 로컬 저장소가 자동으로 생성된다. (git clone)
- 주의해야 할 점은 한 폴더에 하나의 로컬 저장소만 유지해야 한다.

# 2. git add

- commit으로 만들기 원하는 파일만 선택하는 명령어이다.
- git add 뒤에 원하는 `파일 혹은 디렉토리의 이름`을 붙여서 작성한다.

```bash
$ git add <파일 / 디렉토리>
```

- 현재 디렉토리의 모든 변경 내역을 추가하고 싶을 때는 `.` 을 사용한다.

```bash
$ git add .
```

- 작업 디렉토리의 모든 변경 내역을 추가하고 싶을 때는 `-A` 을 사용한다.
  - `.` 과의 차이점은 `.` 은 현재 디렉토리의 변경 내역만을 포함하기에 상위 디렉토리의 변경내역은 포함하지 않지만 `-A` 의 경우 로컬 저장소의 모든 디렉토리의 변경 내역을 포함한다.

```bash
$ git add -A
```

# 3. git commit -m '~~'

- 실제로 commit을 만드는 단계로, -m 뒤에 오는 공간에 해당 commit에 관한 코멘트를 추가한다.

```bash
$ git commit -m '테스트'
```

- commit한 내역은 git log 명령어를 통해 조회할 수 있다.

```bash
$ git log
```

# 4. git remote add

- 로컬 저장소와 원격 저장소를 연결해주는 명령어로, add 다음에 오는 이름으로 올리며, URL에는 일반적으로 github 주소가 들어간다.

```bash
$ git remote add origin https://github.com/아이디/이름.git
```

- origin으로 사용하는 것이 일반적이다.

# 5. git push

- 로컬 저장소에서 작성한 커밋을 원격 저장소로 옮기는 명령어이다.
- remote add에서 만들었던 원격저장소 이름 뒤에 branch 명을 입력한다. (기본 branch는 master)

```bash
$ git push origin master
```

# 다른 사람이 만든 저장소 받아오기

- 원격 저장소를 내 컴퓨터에 받아오기 : Clone
- clone한 원격 저장소가 업데이트 됐을 때 업데이트 된 버전으로 받아오기 : Pull

# 1. git clone

```bash
$ git clone https://github.com/아이디/이름.git
```

- 복사하고자 하는 폴더에서 해당 명령어를 실행하면 원격저장소의 코드들이 복사되며 로컬저장소 폴더가 생성된다. 이 때 폴더 이름은 github에서 설정한 repository 이름으로 생성된다.
- 만약 새로운 폴더가 아닌, 파일들만 해당 폴더에다 옮겨오고 싶을 경우 명령어 뒤에 `.` 을 추가하면 된다.

# 2. git pull

```bash
$ git pull origin master
```

- master branch에서 변경사항을 pull(당겨온다)한다는 것을 의미하며, 원격 저장소의 최신 상태를 로컬 저장소에 업데이트 해준다.
